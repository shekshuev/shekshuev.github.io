### Вариант 8

**1. Какова основная особенность функции COUNT(имя_столбца) по сравнению с COUNT(\*)?**

а) `COUNT(*)` считает только уникальные строки, а `COUNT(имя_столбца)` — все подряд.

б) `COUNT(имя_столбца)` игнорирует строки, в которых данное поле содержит значение `NULL`.

в) `COUNT(имя_столбца)` работает исключительно с числовыми типами данных.

г) Никакой разницы нет, оптимизатор СУБД выполняет их идентично.

**2. Какое поведение характерно для FULL OUTER JOIN?**

а) Он возвращает декартово произведение обеих таблиц (каждая строка с каждой).

б) Он возвращает только те строки, которые имеют `NULL` хотя бы в одном из столбцов.

в) Он возвращает все строки из обеих таблиц; при отсутствии совпадений связанные столбцы заполняются `NULL`.

г) Он требует обязательного использования трех и более таблиц в запросе.

**3. Если в запросе одновременно присутствуют GROUP BY и ORDER BY, в какой момент происходит сортировка?**

а) Сортировка (`ORDER BY`) выполняется до группировки (`GROUP BY`), чтобы подготовить данные.

б) Сортировка выполняется в самом конце, после вычисления всех агрегатных функций и фильтрации `HAVING`.

в) Порядок не имеет значения, СУБД выполняет их параллельно.

г) Сортировка происходит сразу после применения `WHERE`, но до вычисления `COUNT()`.
