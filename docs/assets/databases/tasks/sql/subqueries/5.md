**1. Как сервер базы данных по умолчанию обрабатывает значения `NULL` при сортировке (ORDER BY) по возрастанию (ASC)?**

а) Игнорирует строки с `NULL` и исключает их из итоговой выборки.

б) Размещает строки с `NULL` в самом начале списка (перед минимальными значениями).

в) Размещает строки с `NULL` в самом конце списка (после максимальных значений).

г) Значения `NULL` приравниваются к нулю для числовых столбцов и к пустой строке для текстовых, после чего сортируются штатно.

**2. Какое правило строго регламентирует использование столбцов в блоке SELECT при наличии предложения GROUP BY?**

а) В `SELECT` можно указывать любые столбцы из исходной таблицы, СУБД автоматически выберет первое попавшееся значение для группы.

б) В `SELECT` могут находиться только те столбцы, которые явно перечислены в `GROUP BY`, а также агрегатные функции.

в) В `SELECT` запрещено использовать математические операции над столбцами, если они не включены в `GROUP BY`.

г) В `SELECT` обязательно должна присутствовать функция `COUNT()`, иначе `GROUP BY` не сработает.

**3. За что отвечает конструкция `PARTITION BY` внутри предложения `OVER` при использовании оконных функций?**

а) За фильтрацию строк до применения оконной функции (аналог `WHERE`).

б) За разделение исходного набора данных на логические окна (группы), внутри которых функция производит независимые вычисления.

в) За физическое партицирование (разделение) таблицы на жестком диске для ускорения запроса.

г) За порядок сортировки строк внутри окна вычислений.
