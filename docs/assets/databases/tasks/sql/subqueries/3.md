### Вариант 3

**1. Какое из следующих утверждений точно описывает ограничение PRIMARY KEY (Первичный ключ)?**

а) Допускает наличие значений `NULL`, если таблица связана с другой таблицей.

б) Гарантирует уникальность записи и не допускает наличия значений `NULL`.

в) Может состоять строго из одного столбца (поля) таблицы.

г) Автоматически создает связь с любой другой таблицей в базе данных.

**2. Как ведет себя оператор INNER JOIN при объединении двух таблиц?**

а) Возвращает только те записи, для которых найдено соответствие ключей в обеих объединяемых таблицах.

б) Возвращает все записи из обеих таблиц, заменяя отсутствующие данные на `NULL`.

в) Возвращает только те записи, которые не имеют совпадений в связанной таблице.

г) Требует обязательного использования агрегатных функций.

**3. В чем заключается фундаментальное отличие оконных функций от агрегатных (в связке с GROUP BY)?**

а) Оконные функции уменьшают (схлопывают) количество возвращаемых строк до одной на каждую группу.

б) Оконные функции вычисляют результат набора строк, связанного с текущей строкой, сохраняя при этом исходное количество строк в выборке.

в) Оконные функции могут применяться только к числовым типам данных.

г) Оконные функции не требуют указания предложения `OVER()`.
