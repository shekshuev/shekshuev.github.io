### Вариант 2

**1. Какое из утверждений корректно описывает разницу между предложениями WHERE и HAVING?**

а) `HAVING` выполняется до `GROUP BY` и фильтрует исходные строки таблиц.

б) `WHERE` может использовать агрегатные функции (например, `SUM`, `COUNT`) для фильтрации данных.

в) `WHERE` фильтрует отдельные строки до группировки, а `HAVING` фильтрует итоговые результаты после применения `GROUP BY`.

г) Использование `HAVING` возможно только при одновременном наличии предложения `ORDER BY`.

**2. В чем заключается ключевое отличие оператора UNION ALL от оператора UNION?**

а) `UNION ALL` объединяет столбцы двух таблиц, а `UNION` — строки.

б) `UNION ALL` сохраняет все дублирующиеся строки из объединяемых выборок, а `UNION` удаляет дубликаты.

в) `UNION` автоматически сортирует итоговую выборку по убыванию, а `UNION ALL` — по возрастанию.

г) Никакого отличия нет, это синонимы в стандарте SQL.

**3. Что возвращает оператор RIGHT JOIN?**

а) Декартово произведение двух таблиц (каждая строка с каждой).

б) Только те строки, для которых найдено точное совпадение ключей в обеих таблицах.

в) Все строки из левой таблицы и только совпавшие строки из правой.

г) Все записи из правой таблицы, а для левой таблицы — совпавшие записи либо значения `NULL` при отсутствии совпадений.
