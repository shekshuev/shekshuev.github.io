**1. Что произойдет при выполнении оператора CROSS JOIN между таблицей А (5 строк) и таблицей Б (10 строк)?**

а) Запрос вернет 5 строк, отбросив лишние записи из таблицы Б.

б) Запрос вернет 15 строк (результат сложения строк).

в) Запрос вернет 50 строк (декартово произведение, каждая строка А соединится с каждой строкой Б).

г) Запрос завершится ошибкой, так как для `CROSS JOIN` не указано условие связи `ON`.

**2. Допустимо ли использование математических операторов внутри агрегатных функций? (Например: `SUM(price * quantity)`)**

а) Нет, агрегатные функции принимают только один чистый столбец.

б) Да, СУБД сначала выполнит математическую операцию для каждой строки, а затем произведет агрегацию полученных результатов.

в) Да, но только если в запросе отсутствует предложение `GROUP BY`.

г) Нет, математические вычисления можно производить только после применения агрегатных функций (например, `SUM(price) * SUM(quantity)`).

**3. В чем ключевое отличие оконной функции RANK() от ROW_NUMBER()?**

а) `RANK()` работает только с числовыми данными, а `ROW_NUMBER()` с любыми.

б) При совпадении значений сортировки `RANK()` присваивает строкам одинаковый ранг и пропускает следующие номера, а `ROW_NUMBER()` всегда присваивает строкам строго уникальные последовательные номера.

в) `ROW_NUMBER()` требует обязательного указания `PARTITION BY`, а `RANK()` работает без него.

г) Никаких отличий нет, это синонимы.
